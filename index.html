Stay & Fly –
2nd Place Master Project at ESME

Stay & Fly est une plateforme de voyage Web3 complète, développée par trois étudiants ingénieurs de dernière année à l'ESME. L'objectif était de repenser la réservation de voyages avec les technologies blockchain, en partenariat avec Amadeus.

  • Réservation d'hôtels + revente : les réservations sont transformées en NFT (via smart contracts ERC-721) et peuvent être revendues sur une marketplace privée. Les hôtels ont une période de rachat prioritaire.
  • Négociation de vols : l'utilisateur propose un prix, les compagnies peuvent l'accepter ou proposer un contre-offre via une interface dédiée.
  • Planification d’activités : grâce à Ticketmaster et Google Maps, l’utilisateur construit son itinéraire personnalisé avec des concerts, événements, etc.

Architecture full-stack avec React, Node.js, SQL Azure, Web3.js, blockchain privée Ethereum (Geth). Sécurité assurée par JWT et paiements sécurisés.

HelpDesk AI – Chatbot interne pour DSI

HelpDesk AI est un assistant virtuel développé en Python pour aider les équipes internes d'une DSI à répondre automatiquement aux demandes des collaborateurs.

  • Compréhension du langage naturel : grâce à LangChain, le bot comprend les questions courantes comme "Comment accéder au VPN ?" ou "Où trouver la charte informatique ?"
  • Réponses basées sur des documents internes : une base de données MongoDB contient les procédures, manuels, et FAQ indexés.
  • API REST légère : FastAPI permet une intégration fluide avec l'intranet ou Microsoft Teams via webhook.

Le projet a été pensé pour automatiser jusqu’à 60 % des demandes récurrentes sans intervention humaine, avec une interface simple à maintenir et enrichir.

SecureMail –
Messagerie chiffrée locale

SecureMail est une application de messagerie conçue pour les environnements internes sensibles (ex. cabinet médical, réseau d'entreprise). Elle permet l’échange sécurisé de messages en local, sans aucun serveur externe.

Les messages sont chiffrés côté client en AES-256 et les clés échangées via Diffie-Hellman. L’interface, développée avec Electron, permet une compatibilité multi-plateforme et un déploiement facile.

Fonctionnalités clés :

  • Chiffrement de bout en bout des messages
  • Historique stocké localement de manière sécurisée
  • Échange de clés sécurisé en local, sans Internet requis
  • Option de destruction automatique des messages sensibles

Stack technique :

  • Electron, Node.js, crypto-js
  • Interface minimaliste et responsive
  • Mode sandbox pour tests hors production

AuthAudit –
Outil d’audit de connexions

AuthAudit est un outil de sécurité offensive permettant de simuler des attaques par dictionnaire sur les services tels que FTP, SSH ou RDP. Il a été développé dans le cadre d’exercices d’audit interne et de sensibilisation à la sécurité des connexions réseau.

Le programme permet de lancer des attaques en local, analyser les réponses des services, générer un rapport d’audit, et simuler des comportements suspects détectables par un SIEM.

Fonctionnalités clés :

  • Détection des ports ouverts et services exposés
  • Attaques personnalisables par dictionnaire
  • Génération automatique de rapports HTML
  • Comportement configurable pour simulation de scénarios d’attaque

Stack technique :

  • Python, Flask, Requests
  • Compatibilité Linux (Kali, Debian, Ubuntu)
  • Interface web minimale pour déclenchement et suivi des attaques
